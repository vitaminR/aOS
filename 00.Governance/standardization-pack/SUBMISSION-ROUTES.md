# Submission Routes and Official Links

This file answers two different questions:

1. **Where does formal engagement happen?**
2. **What is each route for?**

## First, the reality check

There is **no single public one-click ISO button** for “submit my new AI standard.”

For ISO, the normal route is:

- engagement occurs through the relevant **National Standards Body (NSB)**
- that body routes the proposal into the correct committee path
- for U.S.-based participation, that often means **ANSI** involvement

For IEEE, there **is** a real online standards-project system, but it still requires the right project framing and usually a sponsoring standards body / committee.

---

## 1. OSI naming precedent

### Official standard title

- `ISO/IEC 7498-1:1994 — Information technology — Open Systems Interconnection — Basic Reference Model: The Basic Model`
- Official page: <https://www.iso.org/standard/20269.html>

Use this as a naming precedent, not as a template to copy blindly.

## 2. IEEE SA route

### Best use for IEEE

IEEE is the most practical early-stage standards path.

### Key links for IEEE

- IEEE standards development overview: <https://standards.ieee.org/develop/>
- IEEE NesCom overview: <https://standards.ieee.org/about/sasb/nescom/>
- IEEE PAR submitter guidance: <https://standards.ieee.org/about/sasb/nescom/submitter-guidance/>
- IEEE myProject system (PAR submission): <https://development.standards.ieee.org/myproject-web/public/view.html#landing>
- IEEE contact page: <https://standards.ieee.org/about/contact/>
- IEEE AIS initiative page: <https://standards.ieee.org/initiatives/artificial-intelligence-systems/>

### What the IEEE route actually is

In IEEE, the formal first step for a new standards project is a **PAR**:

- **PAR = Project Authorization Request**
- it defines the **scope**
- it defines the **purpose**
- it defines the **need**
- it must be reviewed before a formal project starts

### Typical first steps in IEEE

- review PAR guidance
- contact IEEE / relevant initiative contact if needed
- refine the proposal into a standards-project shape
- only then consider a formal PAR

### Direct submission status for IEEE

Formal PAR submission occurs in the IEEE `myProject` system and requires an authorized submitter account.

## 3. ISO / ANSI route

### Best use for ISO / ANSI

ISO is best pursued after the scope is cleaner and external support is in place.

### Key links for ISO / ANSI

- ISO developing standards overview: <https://www.iso.org/developing-standards.html>
- ISO get involved page: <https://www.iso.org/get-involved.html>
- ISO/IEC JTC 1 page: <https://www.iso.org/committee/45020.html>
- ISO/IEC JTC 1/SC 42 page: <https://www.iso.org/committee/6794475.html>
- ISO applications portal: <https://login.iso.org/>
- ANSI (U.S. NSB and secretariat route from the committee pages): <https://ansi.org/>

### Most relevant committee home

For an AI terminology / reference model / ontology proposal, the most likely home is:

- **ISO/IEC JTC 1/SC 42 — Artificial intelligence**
- especially **WG 1 — Foundational standards**

### Important note

ISO’s own guidance says standards are driven through market need and national-body participation.
In practice, this usually means:

- do **not** just upload a concept publicly and wait
- work through the national body and committee structure

### Direct submission status for ISO / ANSI

Formal ISO participation and submission require human participation, accounts, and typically national-body engagement.

## 4. NIST route

### Best use for NIST

NIST is best for:

- alignment
- credibility
- public-comment style engagement
- showing how the model complements existing AI risk and governance frameworks

### Key links for NIST

- NIST AI RMF overview: <https://www.nist.gov/itl/ai-risk-management-framework>
- NIST AI RMF Playbook: <https://airc.nist.gov/AI_RMF_Knowledge_Base/Playbook>
- NIST AI RMF crosswalks page: <https://www.nist.gov/itl/ai-risk-management-framework/crosswalks-nist-artificial-intelligence-risk-management-framework>
- NIST AIframework contact: `AIframework@nist.gov`

### What the NIST route actually is

NIST is not the same as formal ISO/IEEE standardization, but it matters because:

- it is respected
- it is widely used in AI governance discussions
- a good crosswalk makes your proposal easier to understand and harder to dismiss

### Direct submission status for NIST

NIST engagement is best handled through prepared crosswalks, comment-ready materials, and direct external communication by the submitting organization.

---

## Recommendation summary

### Recommended practical order

1. **Prepare neutral spec package**
2. **Build AI RMF crosswalk**
3. **Start IEEE outreach / readiness**
4. **Approach ISO through ANSI / national-body route after traction**

This is the least painful realistic path.
